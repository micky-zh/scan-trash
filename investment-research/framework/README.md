# 投资研究框架

本目录保存可迭代的个股研究体系。目标不是输出漂亮报告，而是让每一次投资判断都能被追踪、验证和反思。

## 核心原则

1. 先判断是不是好生意，再判断价格是否便宜。
2. 财务分析顺序优先看资产负债表和现金流量表，再看利润表。
3. 每个重要结论必须有证据、假设、反证条件。
4. 不懂的公司不强行估值，不在能力圈外下注。
5. 报告必须标记日期，后续跟踪必须回看原始假设。

## 文件索引

- `01-投资原则.md`：长期原则、能力圈、行为纪律。
- `02-个股分析流程.md`：标准分析顺序和输出要求。
- `03-财务三表分析.md`：资产负债表、现金流量表、利润表的分析框架。
- `04-生意质量与护城河.md`：行业、商业模式、竞争优势、管理层。
- `05-估值与安全边际.md`：估值方法、情景分析、买入价格。
- `06-风险与卖出标准.md`：反证条件、风险监控、卖出规则。
- `99-待完善框架.md`：新增、更正、未完成的框架内容。

## 报告归档规则

每只股票单独建目录：

```text
investment-research/reports/股票代码-公司名/
├── YYYY-MM-DD-init-深度分析.md
├── YYYY-MM-DD-quarter-tracking.md
└── thesis-log.md
```

命名要求：

- 初始深度报告：`YYYY-MM-DD-init-深度分析.md`
- 季度跟踪：`YYYY-MM-DD-q1-tracking.md`
- 年报复盘：`YYYY-MM-DD-annual-review.md`
- 重大事件复盘：`YYYY-MM-DD-event-事件名.md`
