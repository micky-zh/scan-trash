# Screening Change Log

## Purpose

这份文件专门记录筛选准则的每一次修改。

规则可以变，但不能无痕变化。

## Change Entry Template

复制下面模板追加新记录：

```md
## YYYY-MM-DD | Version X.Y

- 更新时间:
- 更新内容:
- 为什么改:
- 改好在哪里:
- 教训是什么:
- 受影响规则:
- 受影响案例/股票:
- 后续观察项:
```

## 2026-04-27 | Version 0.1

- 更新时间: 2026-04-27
- 更新内容: 建立第一版筛选准则总文档和变更日志制度。
- 为什么改: 之前只有规则文件和零散说明，没有正式的准则台账，无法持续迭代。
- 改好在哪里: 现在规则修改有固定位置、有固定字段，也把“好的是什么、教训是什么”纳入了正式记录。
- 教训是什么: 如果没有变更日志，规则会慢慢演变成凭感觉修补，最后无法解释为什么这样筛。
- 受影响规则: 全部筛选规则。
- 受影响案例/股票: 暂无具体股票案例，本次是制度建设。
- 后续观察项: 下一轮把银行、保险、周期股的行业例外纳入独立记录。

## 2026-04-27 | Version 0.2

- 更新时间: 2026-04-27
- 更新内容: 引入正式黑名单机制，并把“统一估值阈值”降级为仅适用于少数行业的局部信号。
- 为什么改: 全市场静态 `PB`/`PE` 阈值会误导地产、互联网和周期股；同时需要一套可持续维护的排除列表。
- 改好在哪里: 现在可以把明确不想扫描的对象持续排除，且行业差异被正式写入筛选准则。
- 教训是什么: 如果不区分行业，筛选系统会持续制造伪低估；如果没有黑名单，同类噪音会反复进入扫描结果。
- 受影响规则: 基础筛选逻辑、行业估值解释、扫描排除逻辑。
- 受影响案例/股票: `阿里巴巴-WR` 作为重复证券类型样本，已进入默认黑名单。
- 后续观察项: 继续补银行、保险、地产、互联网的细化规则，并把“按证券类型排除”系统化。
