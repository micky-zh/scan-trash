# Sector Screening Framework

## Purpose

这份文件定义“先总筛、再分行业”的筛选方法。

核心原则：

1. 全市场统一阈值只能做垃圾过滤。
2. 真正的估值判断必须在行业内部完成。
3. `PB`、`PE`、股息率都不是通用真理，只是局部信号。

## Layer 1: Global Trash Filter

这一层只做排除，不做估值结论。

建议只看：

1. 极低价股
2. 极低流动性
3. 长期亏损
4. 数据缺失严重
5. 已知不想研究的黑名单对象

## Layer 2: Sector Classification

进入下一层前，先分行业：

1. 银行
2. 保险
3. 地产
4. 互联网/平台/软件
5. 消费品牌/零售
6. 公用事业/电信
7. 周期资源
8. 工业制造/出口

## Layer 3: Sector-Specific Evaluation

### 银行

主要看：

- `PB`
- `ROE`
- 不良率
- 拨备覆盖
- 资本充足率

不要只看低 `PB`。

### 保险

主要看：

- 承保纪律
- 综合成本率
- 浮存金质量
- 投资组合稳健性
- `ROE`

### 地产

主要看：

- 净负债
- 债务到期结构
- 销售回款
- 土储质量
- 现金流

地产长期 `PB < 1` 不足以说明便宜。

### 互联网/平台/软件

主要看：

- 自由现金流
- 现金及回购
- 主营业务护城河
- 监管风险
- 用户粘性

账面净资产往往不是关键，`PB` 参考价值有限。

### 消费品牌/零售

主要看：

- `ROIC`
- 毛利率稳定性
- 现金流转换率
- 定价权
- 分红与回购纪律

### 周期资源

主要看：

- 当前利润是否位于周期高点
- 历史中枢盈利
- 资产负债表韧性
- 资本开支纪律
- 股东回报政策

低 `PE` 在这里经常是假便宜。

## Blacklist Principle

黑名单是系统性的排除机制，不是临时记忆。

适合加入黑名单的对象包括：

1. 明确不符合研究边界的证券类型
2. 重复证券或衍生证券
3. 过去多次验证为高噪音对象
4. 你明确决定未来一段时间不再扫描的公司

黑名单应记录：

1. 代码
2. 名称
3. 加入日期
4. 原因
5. 是否仍然有效

## Iteration Rule

任何行业规则和黑名单调整，都必须写入：

- `docs/principles/screening_change_log.md`
- `rules/blacklists/default.yaml`
